# AI学习路线

该学习路线仅代表笔者对于人工智能领域的理解和观点，也代表着笔者的一次“新生“。假设你是一名之前仅受到高中教育，没有接触过计算机相关内容的大一新生，以下内容你只需要按照顺序去学便可以从中获得收获，而对于那些更有经验的人来说，以下内容也可以作为一个 checklist，看看有什么内容是你还没有涉及的，无论你是一名freshman还是junior甚至是senior，希望都能对你有点帮助。

<span id="before"></span>

## 开始之前

笔者认为，在决定一件在大学中要学习的主要技术之前，要认真的考虑好这个技术的利与弊与未来的前途，这将成为你学习这门技术的根本动力和支持你坚持下去的理由。

众所周知，人工智能作为当下最火的领域之一，是计算机科学的一个分支，尽管HNUST的人工智能专业并不是计算机学院所开设，但是依旧是和计算机类的学科最为密切相关的，并且就按照近几年来说，人工智能绝对也是断层式的在计算机科学领域发展最快的技术和最为前沿的技术，并且对现有的计算机类的技术有着颠覆性作用。最近有一句调侃的话说的是：“你们搞大模型的就是码奸，你们已经害死前端兄弟了，还要害死后端兄弟，测试兄弟，运维兄弟，害死网安兄弟，害死ic兄弟，最后害死自己害死全人类。”，这句话在笔者眼中看来，按照笔者对人工智能近期发展的观察，是完全可以做到这一切的。人工智能似乎对计算机类的大多数岗位具有替代性的威胁，这是否引起了你对人工智能领域的兴趣或者使得你想要从事于这个领域的理由呢？

假如我们将学生的大学经历定义为以「成材」为目的为期四年的培养，从功利主义的视角来看，一个合理的衡量标准是毕业后薪资。以年包计算，且不考虑大多数的离群点，常规大厂的入职门槛通常是 985 硕士学历，通过企业实习的方式构建简历，可以获得 400K 到 600K 不等的年包。而对于通过科研方式构建简历，并且在热门领域具有与企业方向匹配的科研成果的科研岗求职者来说，这个薪资通常可能是 700K。对于方向匹配的博士来说，薪资以 1M 起步也并非是一件少见的事情。我们可以说，大多数你听说到的关于 AI 领域当前的热潮以及对应的薪资高涨都绝非谣言，所以即使不管科研理想，想想真金白银，也会让你足够有动力。

可惜的是，笔者认为，作为一名平平无奇的四非大学生，在没有走弯路的情况下，选择人工智能这条路并不会比选择学习前后端有着任何的优势，或者未来发展的可能，也就是说若你想要吃人工智能时代的红利的话，你付出的并不会比其他的岗位少。其实对于计算机所有岗位来说，你所谓的前途并不会和你选择的岗位或者发展的目标有太大的关系，而是和你的个人竞争力息息相关。笔者可以在这里先说出一个结论：对于未来前途的话，选择人工智能会使得你相比于其他岗位的话上限最高同时也是下限最低的。对于HNUST的读者来说，可以肯定的是，若你未来想找人工智能相关的岗位的话，读研或读博是几乎是你唯一的选择，单纯以一名四非本科的身份去碰瓷大厂的人工智能岗位的话，是比较困难的，一个原因是先例比较少，而且必须要有科研经历，当然中小厂的人工智能岗位还是很有希望的。而相同情况下的前后端工作者来说，笔者就见过很多找到很不错的大厂实习及转正的优秀同学。

相反的是，若你已经确定了想要继续攻读研究生甚至博士的道路的话，那么学习人工智能将会带给你很大的优势，这是学习前后端等技术做不到的，就当今的所有科研领域来说，不管是顶刊顶会还是水刊水会，与人工智能相关的科研领域无疑是最好发论文的，就拿笔者个人的经历来说，笔者在大二下学习开始接触科研领域，一直到大三上结束的这半年，笔者发表了一篇CCF B类会议、一篇SCI 2区TOP期刊，及四篇CCF C类会议，若按照HNUST的毕业条件来说，笔者已经足以达到HNUST的博士毕业要求了。另外一个不可忽略的要素在于科研对于保研申请的裨益。在硕士学历似乎已经是求职敲门砖的当下，保研对于大多数读者来说是一条相当有诱惑力且几乎必须的路线。相较而言，考研会浪费你的大量时间，在研究生期间需要漫长的时间进行冷启动，从而有更大概率错失后续实习的黄金时期。对于保研简历来说，科研经历毫无疑问是最具竞争力的要素，并且有利于你申请到更加顶尖的学校。

## 正片开始

朋友，既然你已经选择了这条路的话，并且作为我的同行，在正式学习任何一门学科之前，做好长久的规划是非常重要的，本文所描述的学习路线是笔者在经历了大学三年的摸爬滚打以后总结而出的，在经过认真分析考虑之后得出的一条相对完美的学习路线，而这条路线只是笔者预想中的，这条路线对于读者的个人能力要求较高，可能有一些内容并不切实际，仅作为笔者的个人建议，读者可以适当的降低对自己的要求，但可以保证的是，相信看完会对你有一定的帮助。

就像学习完前后端以后，最终的成果是开发一个含有多功能、甚至于多平台的网页或小程序APP一样，相信你也想在学习完人工智能以后，能够自己开发出一个类似于豆包的大模型、Agent或者一个集成主流深度学习技术的一个宏大的项目。但这条路还是非常漫长的，需要依靠你一步一步的脚印去实现，虽然说现在的一些如ChatGPT等大模型有一定能力可以帮你直接实现这个最终的梦想，但是笔者还是希望你能够通过自己去实现，这绝对是一件非常有意义的事情。当今的人工智能领域似乎分为很多分支，但是就笔者看来，主流的分支还是分为计算机视觉和自然语言处理。而在开始正式学习这两个大模块之前，需要打好足够的基础。

<span id="math"></span>

### 数学基础

最开始的开始，对人工智能关系最为密切的数理学科是高等数学和线性代数，希望你能够在大一的课堂上好好学习这两门极其重要的学科，作为你基础中的基础，并且务必要保证在期末考试中考到90分以上，这对你绩点的维持有很大帮助。同时可以看一些B站的课程作为辅助，在这里推荐[宋浩数学课程](https://space.bilibili.com/66607740)，通过看这个可以帮你更好的理解课堂上未曾理解的一些知识，若你想获得生动形象的感悟，在这里推荐[3Blue1Brown 的相关讲解](https://www.youtube.com/playlist?list=PLZHQObOWTQDPD3MizzM2xVFitgF8hE_ab)。从本质上来说，你只需要了解这三门课程中的部分内容，即高等数学中多维的积分以及微分、线性代数中矩阵的乘法与求逆以及概率论中的贝叶斯相关内容，从事实上就已经可以读懂大多数科研的论文以及思想了。当然，对于一些涉及数学更多的研究领域，你需要专门了解这个领域，问问领域中的专家，特定地学习更多的数学知识。但是我们毕竟不是数学专业的学生，只是希望在高层建筑上去用到数理知识的小部分核心思想即可，若不是特定对于数学要求较高的科研方向，需要用到很多的公式推导的话，一些这样的最基础的数理知识浅尝辄止即可。

<span id="cs"></span>

### 计算机基础

对于一名刚进入大学的freshman来说，没有系统接触并了解电脑是一件再正常不过的事情，虽然说人工智能领域的发展完全离不开所掌握的算力资源，若你想要训练模型是非常依靠一张好的GPU显卡的，但是在笔者看来，你并不需要去为此购买一台拥有GPU的电脑，这是因为对于大多数畅销的7000-9000的笔记本电脑都配置的只是5060显卡，若你没有打游戏的需求的话，不建议购买像这样的游戏本，轻薄本足够日常使用了，而对于训练模型来说，有很多算力市场如[AutoDL](https://www.autodl.com/home)可以进行显卡的租借。此外，对于电脑小白来说，一般还是建议首先使用 Windows 电脑，本地的办公环境还是要保证一定的轻便性，而由于在此之前绝大多数的读者对电脑的使用环境均是在 Windows 下进行，并且网络上对于 Windows 的资料远大于其他系统，因此在绝大多数情况下，笔者均建议电脑小白选择 Windows 作为自己第一台电脑的操作系统。而对一些开发的技术熟练掌握之后，可以考虑使用Mac，并且建议再安装Linux作为双系统，以进一步提高生产力，并且贴合实际的开发环境，此外，购买电脑首选线上购买，不推荐线下购买电脑，即便你有一些connection。对于计算机的其他基础，如果没有影响你的正常学习和开发的话，或许可以不用了解太多，不过笔者还是推荐学习一下你[缺失的那门计算机课](https://www.criwits.top/missing/)，来对计算机有一个全面的认识，从而加深你对计算机的理解，以免出现一些低级知识的掌握空白。

<span id="tools"></span>

### 学习工具

在这一部分，最重要的也就是Chat with LLM，在当今时代，是否能够好好利用大模型，甚至成为了学习计算机科学中的一个决定性因素。大模型出现之前学习一门新语言或者写一个项目有很多方法，你可能需要查阅文档或者论坛去找到一些在浅层搜索中难以发现的使用逻辑，然后再进行漫长的代码编写和 debug，其中可能还包括大量的重复性工作，但是大模型在出现之后，一切逻辑都被改变了。笔者对此深有体会，大模型对你的学习效率的提升不仅仅是多少百分比这么简单，更是对你自身生产力的一次重大革命。

因此，如何选择一种大模型进行使用成为了比较重要的问题，笔者在经过自身的一些使用体验后，在这里还是推荐国外开发的大模型，而在这里首选[Gemini pro](https://gemini.google.com/)，若你无法进入该网站，需要在线下求助你的相关同学或者前辈，请教如何进行科学上网。虽然pro版的Gemini需要你额外支付一定的费用，但在笔者看来是非常值得的，并且可以去咸鱼获取一定的优惠价格。若你要进行高深的科研代码编写的话，可以考虑ChatGPT的plus甚至是pro，而在其他情况下，Gemini则足以应对。若要推荐国内模型的话，则Qwen或者glm可以作为首选，也可以考虑deepseek，尽管他有着较高的幻觉率。此外，除非你有娱乐性的需求，不建议使用豆包。

另一部分，笔者在这里推荐[Visual Studio Code](https://code.visualstudio.com/)作为你唯一的一款IDE，无论是使用什么语言。假如读者刚刚开始接触编程，可能会使用学校推荐的[CodeBlocks](https://www.codeblocks.org/)或者[DevC++](https://www.dev-cpp.com/)作为你编程C语言与C++的IDE选择，或者其他的[Visual Studio](https://visualstudio.microsoft.com/) 以及 [CLion](https://www.jetbrains.com/clion/)。而当你要进行其他语言的编写的时候，如java或python，那么你就会下载[IDEA](https://www.jetbrains.com/idea/)和[PyCharm](https://www.jetbrains.com/pycharm/)，但是不得不说的是，这么做是非常低效的，而且会占用你大量的内存，按照程序员的角度，这是不可取的。要是用某一个词汇去形容 VSCode，那么文本编辑器可能是比较贴切的之一，VSCode 并不支持任何的编程语言，也就是在你下载了它之后，你并不能直接在里面编辑并且运行 C，也不能直接运行 Python，本质上它只是一个文本编辑器而已，而恰好我们说的这些编程语言的文件都是直接的文本，只是后缀名不同。

VSCode 实际的工作原理是将你本地配置好的环境和它自身的插件结合在一起，你在本地具有 Python 环境，同时你在 VSCode 中安装了 Python 插件，那么你就可以具有直接运行程序，实现代码高亮，错误提示在内的一系列功能。

诸如此类，我们不难想象这样一个场景，当我们在本地分别为各种语言配置好了环境，并且为我们的 VSCode 安装了插件，那么一个兼容无数编程语言的强大 IDE 就诞生了，而且还很简洁。当然假如你有额外的需求，也不用担心，再安装一个插件就好，VSCode 具有极其庞大的社区，开源社区为其制作了无数的插件，可以覆盖你的绝大多数需求。一个好消息是，现在甚至可以将大模型通过插件部署到VScode中，而且性能依然优秀，在这里推荐ChatGPT的[Codex](https://chatgpt.com/codex)和[Claude Code](https://claude.com/product/claude-code)可以部署到VScode上，并且基于该模型一定的权限，使模型有权限可以读懂你的项目文件。

同时，伴随着大模型在代码编程领域的深度应用，代码补全插件应运而生，也就是在你输入了几行代码之后，模型会尝试理解并且预测你将来想写的程序，并且给出提示，此时你只需要按下 Tab 键就可以把它的提示应用到你的文件中。不过笔者在体验了诸多 VSCode 提供的代码补全插件之后，在这里推荐[Cursor](https://www.cursor.com/)。

在学习的过程中，笔者推荐你可以随手进行笔记的记录，在正式学习某些专业知识或者编写程序之前，了解并学会使用 Markdown 可以说是十分重要的技能。通过 [Markdown 官网](https://markdown.com.cn/intro.html)，读者可以了解 Markdown 的全部语法，虽然说是全部，但实际上也非常简单，基本上两三次之后就可以熟练使用。尽管这些标记符号十分清晰，一个可以实时将这些内容渲染为可视化的文档的软件依然有必要存在，在这里推荐 [Typora](https://www.typora.io/)，作为专业的 Markdown 编辑工具，可以从外面导入不同的外观模板，同时对于导出 PDF 可以做到所见即所得。

<span id="python"></span>

### 人工智能语言

对于深度学习及人工智能的研究者及从业者来说，选择python是你唯一的选择，因为笔者还从未看见用相关论文或项目代码不是用python来实现的。虽然C语言或者C++是你现在已经非常熟悉的语言，但是想要学习ai的话，你不得不另外再学习一门python语言。此外，对于学习acm类算法来说，你依然可以使用C++进行编程，因为同时掌握两门语言对于程序员来说并不是稀奇的事情。幸运的是，学习python并不会花费你太多精力，对于笔者来说，并没有进行python的具体的系统性学习，那么应该如何学习python呢，首先，你需要对python为什么是一门人工智能必用语言有着一定的了解。

在此基础上，我们有必要简单描述一下在主流深度学习领域中对于编程部分的技术栈，在领域的早期，各种的学习框架以及各类内容几乎是百花齐放，但是伴随着几年的迭代，社区趋于稳定，框架的选择也逐渐收敛。我们使用 MiniConda 或者 uv 进行环境管理，PyTorch 进行模型的搭建以及训练框架的搭建，对于更加现代的领域，诸如 LLM, VLM 或者 VLA，我们均使用 Transformers 库，这是一个 Huggingface 搭建的对于模型进行进一步封装的库。PyTorch 和 Python 的关系，简单来说就是：Python 是一门编程语言，而 PyTorch 是基于 Python 构建的专门用于深度学习的工具包（第三方库）。而且只有PyTorch具有现在最强大的模型代码库，若用其他语言的话，可能需要自己手写模型的全部代码，这是一件非常头疼的事情。在此基础上，如果你对于 LLM 以及 VLM 存在一些部署的需求，目前的主流框架是 vLLM。如果你不了解 Huggingface，这是一个面向模型的 Github 类似的内容，使用者可以上传自己的模型权重或者数据集（数据集的概念相当广泛，这几乎就是一个网盘）到 Huggingface，以实现开源以及全球的分发，除此之外 Huggingface 也形成了全球最大的人工智能尤其是模型社区。

回到具体的python学习，并没有比较推荐的网课，因为笔者认为该语言非常简单，是你在学习完C和C++以后自然而然就能够靠自己的本领掌握的一门语言，读者可以根据教程配置好开发环境后，直接询问大模型并且获得答案，从能能够一步步地掌握python的具体语法。市面上的一些网课可能时间非常长，而且性价比不高，有些地方也没必要掌握的那么细节。

<span id="mlcv"></span>

### 机器学习与传统计算机视觉

笔者可以简单给出结论，机器学习以及传统的计算机视觉在大多数时候并不会起到什么作用，或者说这些内容已经通过理论完善，并且具有一定的上限。尽管有的时候可以为你设计新的算法带来启发，但是并不在我们学习的最短路线中，不过这里给出参考资料。

对于系统学习机器学习的书籍，相较于网络较火的西瓜书，也就是周志华老师的《机器学习》，笔者更加推荐李航老师的《统计学习方法》。而传统的计算机视觉相关的内容，则推荐冈萨雷斯的《数字图像处理》。对于网课的学习不是很有必要，因为笔者认为你只需要懂得机器学习中的核心算法即可，如果非要推荐网课的话，可以观看吴恩达老师主讲的[cs229](https://www.youtube.com/watch?v=jGwO_UgTS7I&list=PLoROMvodv4rMiGQp3WXShtMGgzqpfVfbU)，不过它和偏基础的入门课不太一样，CS229 的数学推导会硬核很多，对线性代数和概率论的要求比较高，如果你数学基础比较好，而且想把算法底层的数学原理抠得很明白，那这门课就很适合你，不然刚入门直接看可能会非常折磨。我们还是建议你学习一些最经典算法即可，之后可以去 Kaggle 跑个泰坦尼克号之类的新手项目，多动手跑跑数据很快就能摸到门槛。

同时，尽管这些内容有的时候依然实用，但是多半在实际的使用中，我们都是以使用者的需求而非创新性的追求来看待。好消息是 Python 的众多库，如机器学习的 scikit-learn 以及计算机视觉的 OpenCV，其实都支持了这些算法，因此往往只需要简单向大模型进行提问，并且调动这些就可以实现对应的功能。

一些反对者或者保守主义者不免担心，跳过了这些打基础的环节，是否会导致将来自己学习一些东西的时候不太牢靠。尽管在笔者的视野中并没看到这样的案例，但是进行适当的学习倒也无所谓，只是看到互联网上繁多的课程，没有必要因此看花了眼，选中几个课程并且进行快速的学习即可。想当初笔者在视频网站中比较不同视频合集介绍算法的异同点并且都看了一遍，如今回过头来看，除了浪费时间，没有得到任何的帮助。尽管传统机器学习与计算机视觉方法在当下并非主流，但在实际工程应用中依然有着广泛的使用，例如工业检测、图像处理、边缘设备部署等场景；而在理论研究领域，它们更是构建现代 AI 的基础。如果你对工程落地或算法原理感兴趣，这些内容仍然值得系统学习。

### 深度学习

在开始深度学习的学习之前，相信你已经对于人工智能领域拥有了一定的基础，知道了神经网络及模型权重相关的一些核心概念。到了深度学习的领域，你正式开始入门人工智能这门学科了，需要更深度的理解各个方面的知识，并且融会贯通。当然，这只是一个开始。

由于人工智能时代的红利和绝大多数课程的商业化教程，现在市面上有太多质量不一，且需要付费的大量教程，而在这里给出我们认为的非常权威而且亲身实验过课程，在这里强推荐你观看斯坦福大学的[CS231n](https://www.youtube.com/playlist?list=PLoROMvodv4rOmsNzYBMe0gJY2XS8AQg16)，这门经典的计算机视觉课程现如今已经涵盖了对于计算机视觉以及泛深度学习中的绝大多数基础内容，对于初学者来说绝对是最好的教材。无论你是否擅长英语，这门课程也都依然推荐，请在磨砺你的英语水平的同时观看，顺带一提，YouTube 提供了实时字幕翻译的功能。与此同时，假如说一定要选择一个中文教程，在这里照例推荐李沐的[动手学深度学习](https://www.bilibili.com/video/BV1if4y147hS/)，同时并不建议观看其中任何的动手部分以及比赛部分，而且快速过掉全部知识点。至此你便了解了深度学习的基础知识，可以投身于特定领域以及代码的学习了。而如果你对这两门课程的学习有点吃力的话，也可以先学习小土堆的[PyTorch深度学习快速入门教程](https://www.bilibili.com/video/BV1hE411t7RN/?spm_id_from=333.337.search-card.all.click)作为一个微小的过渡。

与此同时，当你学完如今计算机视觉及深度学习的技术之后，可以进入大模型相关课程的学习了，作为人工智能核心两大领域的另一个分支，也是当今适用面最广的一个技术。在这里强推荐 [CS336](https://www.youtube.com/playlist?list=PLoROMvodv4rOY23Y0BoGoBGgQ1zmU_MT_)，这些内容看上去更加的专业，需要更多的时间仔细学习。在这里笔者建议读者自行实现几个程序，比如说经典的 MNIST 识别数字，或者可以手动实现一个 ViT 来识别 ImageNet，这个环节可以在AI的辅助下完成，来确保自己已经可以熟练融入编码中带有AI协助的工作流程。不过必须说明的是，这些代码说到底只是练手而已，而实际的项目往往从需求中产生，在此之前先学一学如何了解一个新领域吧。
